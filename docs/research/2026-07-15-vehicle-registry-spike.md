# Research spike: «Додати авто за держномером» — джерела даних

**Дата:** 2026-07-15
**Автор:** research spike (Claude)
**Питання:** чи можна дозволити користувачу ввести лише держномер (напр. `BC1234HX`) і автоматично заповнити VIN, рік, двигун, тип пального і **дату закінчення ОСЦПВ**?
**Закриває:** `docs/FEATURE-MAP.md` — «Додавання авто за держномером … **СПОЧАТКУ research-spike**»

---

## TL;DR — вердикт

| Частина фічі | Вердикт |
|---|---|
| Номер → VIN, марка, модель, рік, об'єм двигуна, **тип пального** | 🟢 **Технічно і юридично можливо.** Opendatabot `/v3/tech-passport` дає всі ці поля. **8 100 грн/рік** за 18 000 запитів (опублікована ціна в оферті) |
| Номер → **дата закінчення ОСЦПВ** | 🔴 **НЕМОЖЛИВО. Джерела не існує.** Перевірено всі 40 ендпоінтів API МТСБУ + всі 74 ендпоінти Opendatabot |
| Номер → VIN безкоштовно (пілот) | 🟡 baza-gai: безкоштовно 1000 запитів/міс, але **VIN лише для реєстрацій з 2021 р.**, нема ToS, нема юрособи |
| VIN → двигун/пальне через безкоштовний декодер | 🔴 vPIC порожній на ЄС-авто (перевірено емпірично, див. §5) |

**Рекомендація:** робити фічу, але **без ОСЦПВ** і **не першою**. Спочатку — ручні поля VIN + номер (нуль залежностей), потім пілот на безкоштовному ключі, і лише за підтвердженим попитом — платний Opendatabot.

> ⚠️ **Головна знахідка, яка ламає початкове формулювання фічі:** дату закінчення ОСЦПВ не віддає жодне джерело — навіть офіційний B2B API МТСБУ. Фразу «авто-дата ОСЦПВ → авто-інтервал» треба **прибрати з опису фічі в беклозі**, інакше ми обіцяємо в UI те, чого не існує.

### ⚠️ Найважливіша знахідка по ОСЦПВ

Я перевірив **усі 40 ендпоінтів** опублікованої спеки МТСБУ (`policy-doc.mtsbu.ua/DpApiDoc/doc/api_data.json`, 676 719 байт, завантажено 2026-07-15) на будь-які поля дат чинності:

- `GetStatus/ByPlateNum` повертає: `FContract_DViewStatus_Code` (`Active`/`Canceled`/`Duplicate`/`Renewed`) + `RequestDate` (дата **запиту**, не полісу).
- **Полів `DateFrom` / `DateTo` / періоду дії немає в жодному з 40 ендпоінтів.**
- `GetStatus/ByPlateNum` приймає **лише** параметр `PlateNum` — параметра дати нема, тобто навіть бінарним пошуком «чинний на дату X» дату закінчення не вивести (це вміє лише веб-форма — за Turnstile).
- `GetHistory/ByVIN` — це **бонус-малус** (історія страхових випадків), не дати полісу.
- В Opendatabot **жодного** ендпоінта про страхування: grep по `осцпв|мтсбу|страхув|insurance|policy|поліс` по всіх 74 шляхах → **0 збігів**.

**Висновок: дату закінчення ОСЦПВ не можна отримати автоматично навіть теоретично, навіть за гроші, навіть із повним доступом до МТСБУ.** Максимум, що дає МТСБУ — «поліс зараз активний / неактивний».

**Це не блокер для продукту:** у беклозі вже є **P1/S «Compliance-інтервали за датою: пресети ОСЦПВ, техогляд, зелена карта»** — ручний ввід дати, який їде існуючим Telegram-циклом нагадувань. Фронтенд уже очікує пресет `{title: 'Страховка ОСЦПВ', interval_km: null, interval_days: 365}`. Це і є відповідь на ОСЦПВ. Автоматики тут не буде — і не треба.

## 2. Opendatabot — деталі

**Як знайдено:** `docs.opendatabot.com` — це Swagger UI; спеки лежать у `docs.opendatabot.com/js/config/apiVersions.json` → `https://opendatabot.com/api/v4/openapi/doc`, `https://opendatabot.com/apidocsV3.php`, `https://opendatabot.com/api/v3/private/api-docs-full` (остання — 74 шляхи, 1 966 923 байт). Усі три завантажено 2026-07-15.

> ℹ️ `opendatabot.ua` (веб-сайт) віддає **403** на всі запити — Cloudflare. Ціни/оферту довелось читати через Wayback. **`opendatabot.com` (API + спеки) віддається нормально.**

### Цільовий ендпоінт

```
GET https://opendatabot.com/api/v3/tech-passport?apiKey=<KEY>&number=BC1234HX
```
Тег: «Інформація про авто» · summary: «Отримання інформації по номеру авто» · security: `apiKey` (query-параметр).

Повертає (схема `tech-passport-item`):

| Поле | Приклад | Мапиться на |
|---|---|---|
| `vin` | `SJNFBAF15U6333363` | **нове поле `Car.vin`** |
| `brand` / `model` | `NISSAN` / `JUKE` | `Car.brand` / `Car.model` |
| `makeYear` | `2013` | `Car.year` |
| `capacity` | `1598` (см³, integer) | `Car.engine` (зараз free-text `String(100)`) |
| `fuel` | `БЕНЗИН` | `Car.fuel_type` — **потрібна таблиця мапінгу** |
| `kind` | enum: `ЛЕГКОВИЙ`, `ВАНТАЖНИЙ`, `МОТОЦИКЛ`, … | — |
| `color`, `lastDate`, `operCode`, `wanted`, `number` | | — |

> `/v4/transport` — той самий сенс, але **без `fuel`** і без `wanted`. Для нашої фічі **v3 строго кращий**.

### Ціна (опубліковано в оферті)

`https://opendatabot.ua/open/oferta` — знімок Wayback від **2026-06-25**, прочитано 2026-07-15:

> **п. 5.11. Запити до API перевірок транспортних засобів: Вартість — 8 100 грн · Строк — Рік · Кількість запитів — 18 000**

Для порівняння з тієї ж оферти: п. 5.8 «Перевірка … у веб-інтерфейсі» — 399 грн/міс або 2 999 грн/рік. Споживацька разова перевірка авто на сайті — **99 грн** (`opendatabot.ua/open/check-auto`, знімок 2026-05-05).

> ⚠️ **Обережно:** у пошуковій видачі «99 грн» часто подають як ціну API. Це неправда — 99 грн це разова веб-перевірка для кінцевого користувача. Ціна API — 8 100 грн/рік (п. 5.11). Також у прайсі є «Витяги з ЄДР — 99₴», що плутає ще більше. **Сторінка `/open/api` цін не публікує взагалі** («Прозоре ціноутворення» — і кнопка «Підключитися» → відділ продажу). Єдине опубліковане джерело ціни на transport-API — це оферта, п. 5.11.

### 🔴 Практичний блокер: оплата

- п. 5.15: оплата **карткою** передбачена **лише** для п. 5.8 (веб-інтерфейс).
- п. 5.14: «Послуги вважаються сплаченими після зарахування коштів **на розрахунковий рахунок**».

→ **Пакет API купується за договором через банківський переказ, тобто потрібен ФОП або ТОВ.** Для хостингу «власник тримає для платних користувачів» це реальна умова входу, а не формальність. Якщо ФОП уже є — питання зняте; якщо ні — це окрема задача, більша за саму фічу.

### ToS — що можна і що ні

- ✅ **п. 2.4:** «Клієнт має право використовувати Інформацію на свій власний розсуд, **для своїх власних (у тому числі комерційних) потреб**».
  > ⚠️ Моя інтерпретація (не цитата): «власні потреби» — формулювання, яке однозначно покриває *показ даних власнику його ж авто в нашому UI*. Воно **не** є явним дозволом на перепродаж/ресейл даних як окремого продукту. Ми робимо перше, не друге — але якщо колись з'явиться «перевір будь-яке авто» як платна фіча, це треба узгодити з Opendatabot письмово.
- ✅ **п. 2.2:** API — це штатний, передбачений договором канал.
- 🔴 **п. 5.19 + 5.20:** заборонено автоматизоване зчитування **веб-інтерфейсу** (п. 5.8); штраф **100 000 грн за кожен встановлений факт** + блокування без повернення коштів.
  → **Ніколи не скрейпити opendatabot.ua. Тільки API за ключем.** Обмеження стосується веб-кабінету, не API — але ціна помилки тризначна в тисячах гривень.

## 3. baza-gai.com.ua

- Документація: `https://baza-gai.com.ua/api` (200, прочитано 2026-07-15). Заявка: «Наш API открыт для всех желающих» (`/about`).
- Ендпоінти: `GET /nomer/{plate}`, `GET /vin/{vin}`, `/search?vendor=…&fuel=…`, `/make`, `/plate_fields`, `/taxes`.
- **Auth обов'язковий:** заголовки `Accept: application/json` + `X-Api-Key`. Перевірено емпірично: запит без ключа → **401 `{"message":"Неправильный API-ключ"}`**. Заголовок відповіді `X-RateLimit-Remaining` = залишок на місяць.
- **Безкоштовно: 1000 запитів/місяць** (self-serve ключ). Платні тарифи — buckets 1к/10к/30к/100к/>100к, **ціни не опубліковано ніде**, тільки «по запиту». **Не верифіковано — не вгадую.**
- Повертає `vin, vendor, model, model_year, displacement, kind, color, is_stolen, operations[]`. **`fuel` у відповіді нема** — тільки як фільтр пошуку.
- 🔴 **`vin` доступний лише для реєстрацій з 2021 року** (пряма цитата з доків).
- 🔴 **ПІБ власника не дає** — прямо: «Как узнать Ф.И.О. владельца номера? К сожалению, такой возможности нет». *(це, до речі, **плюс** для нашої приватності — див. §6)*
- 🔴 **ToS / оферти не існує.** Перевірено 10 шляхів — усі 404. Нема ні заборони комерційного використання, ні жодного захисту/SLA для нас.
- 🔴 **Bus factor = 1.** Юрособи не знайдено; за даними iTunes lookup API — розробник `Pavel Hubin` (застосунок оновлено 2026-06-09, тобто проєкт живий).
- 🟡 API-контракт стабільний: останній запис у changelog — `1.1.0 (20 Ноября 2022)`, тобто **ламких змін ~3.5 роки**.
- Історичний сигнал: ключ з'явився реактивно — «В связи с некоторыми пользователями, которые делали сотни тысяч автоматизированных запросов каждый день, ограничиваем наш API» (новина 20.04.21).

**Позиція:** ідеальний **пілотний** інструмент (безкоштовно, self-serve, миттєво), **непридатний як єдина продакшн-залежність** для платних користувачів. Причому обмеження «VIN лише з 2021 р.» для нас **менш болюче, ніж здається**: цільова аудиторія — свіжі ЄС-імпорти, які якраз реєструвались після 2021.

## 4. МТСБУ

- **`mtibu.kiev.ua` мертвий** — віддає дефолтну заглушку IIS (703 байти). Актуальний домен — **`mtsbu.ua`**.
- Публічна перевірка: `https://policy.mtsbu.ua/` (= `policy-web.mtsbu.ua`; `mtsbu.ua/services/perevirka-chinnosti-polisu` → 302 на нього).
  - Пошук за: **держномером**, номером полісу, **VIN**; окремо «Зелена картка»; плюс поле **Дата** («Чинність полісу на обрану дату»).
  - Захист: `<script src="https://challenges.cloudflare.com/turnstile/v0/api.js?compat=recaptcha">` — **Cloudflare Turnstile**, не Google reCAPTCHA. *(прапорець `compat=recaptcha` підсовує `grecaptcha`-сумісний JS API — саме тому наївний grep по «recaptcha» дає хибний збіг; у документації це важливо назвати правильно)*. Плюс ASP.NET `__RequestVerificationToken`. Форма POST на `/`, JSON-ендпоінта нема.
- **Офіційний B2B API існує:** `https://policy-doc.mtsbu.ua/DpApiDoc/doc/index.html`, спека `…/api_data.json` (40 ендпоінтів, версія `5.20180315.0`).
  - Призначення (цитата): «опис web-сервісів для організації автоматичної взаємодії МТСБУ та страховиків — **членів МТСБУ**».
  - Auth: токен **HTTP Basic** у нестандартному заголовку `Authentication:`. **Read-сервіси — без VPN**, write — тільки страховики + VPN МТСБУ.
  - Підключення: «слід звернутися до МТСБУ». Self-serve нема, ціни нема, чи дадуть не-страховику — **не верифіковано**.
  - ⚠️ Усі хости у спеці — **`-test`** (`policy-read-test.mtsbu.ua`). **Продакшн-хост не верифіковано.**
- **І все одно це не вирішує задачу** — дат чинності в API нема (див. §1 ⚠️).
- Комерційні агрегатори (hotline.finance та ін.) фронтять перевірку МТСБУ і заявляють «Джерело даних — офіційна база МТСБУ». Це їхня **маркетингова заява**; правову підставу (членство в МТСБУ? окремий договір?) я **не верифікував**.

| `DisplacementL` | **`''`** 🔴 | `1.781000` |
| `BodyClass` | **`''`** 🔴 | `Hatchback/Liftback/Notchback` |
| **Заповнено полів** | **25 / 154** | **42 / 154** |

`ErrorText` на ЄС-VIN: `14 - Unable to provide information for some of the characters in the VIN, **based on the manufacturer submission**`.

> ⚠️ **Чесне застереження:** мій ЄС-VIN синтетичний, тому там же є `1 - Check Digit does not calculate properly` і `400 - Invalid Characters Present` — це артефакт тестових даних, не властивість vPIC. Але висновок від цього не змінюється: помилка **14** прямо каже, що причина — **відсутність даних від виробника** в базі NHTSA. VW не подає в NHTSA специфікації авто для ринку ЄС. Реальний VIN дасть чистіший check digit, але `Model`/`Fuel`/`Displacement` для німецького Golf лишаться порожні.

**Що з цього виходить (важливо для Плану B):**

1. 🟢 **Рік і марку можна дістати офлайн, без жодного API.** vPIC витяг `ModelYear=2016` і `PlantCountry=GERMANY` чисто зі **структури VIN** (WMI `WVW` + код модельного року в 10-й позиції, `G` = 2016, ISO 3779). Це ~30 рядків коду і таблиця з 30 символів — **нуль залежностей, нуль вартості, нуль правових питань**.
2. 🔴 **Модель / двигун / пальне для ЄС-авто безкоштовний декодер не дасть.** Ці поля живуть у платних каталогах (Vincario), або в українському реєстрі (Opendatabot `fuel` + `capacity`).

## 6. Правовий бік (ключове — читати повністю)

### Первинне джерело: ст. 34-1 ЗУ «Про дорожній рух» № 3353-XII

Прочитано з `https://zakon.rada.gov.ua/laws/show/3353-12/print` (2026-07-15). Цитати:

> «Інформація про зареєстровані транспортні засоби та їх власників, належних користувачів, що міститься у Єдиному державному реєстрі транспортних засобів … **є відкритою та загальнодоступною**.»

> «Для фізичних та юридичних осіб інформація з ЄДРТЗ надається шляхом **пошуку за суб'єктом (власником транспортного засобу)** … **за умови ідентифікації такої особи** з використанням засобів електронної ідентифікації, що мають **високий або середній рівень довіри** …»

> «За запитом посадових осіб органів державної влади … судів, органів Національної поліції, органів прокуратури, СБУ, БЕБ, ДБР та НАБУ, **Моторного (транспортного) страхового бюро України**, адвокатів, нотаріусів, інспекторів з паркування інформація з ЄДРТЗ … надається за суб'єктом … **чи за номерним знаком** …»

**Що це означає для нас — три висновки:**

1. **Дані самі по собі законодавчо відкриті.** Пошук авто за номером не є чимось незаконним за своєю природою — відкритість цих даних прямо написана в законі.
2. 🔴 **Але офіційний пошук ЗА НОМЕРНИМ ЗНАКОМ — це закритий перелік**: держоргани, суди, поліція, **МТСБУ**, адвокати, нотаріуси, інспектори з паркування. **Приватного SaaS у цьому переліку нема і не буде.** Для фізосіб офіційний канал дає пошук **тільки за власником (себе)** і тільки з електронною ідентифікацією (BankID/Дія).
   → Саме тому Opendatabot і baza-gai будують пошук за номером **на опублікованих відкритих даних МВС**, а не на прямому доступі до ЄДРТЗ. Це юридично інший канал, і саме тому їхні дані **відстають** (baza-gai: «Ежемесячно, с обновлением официальных данных МВД»; VIN лише з 2021 р.).
   → Практично: **ми не можемо стати офіційним отримувачем — ми можемо лише бути клієнтом посередника.** Ризик легальності каналу несе посередник, і це аргумент за Opendatabot (юрособа + договір + оферта) проти baza-gai (фізособа, нуль ToS).
3. **Власник може дізнатись, хто дивився його авто:** «За заявою власника … орган реєстрації … надає інформацію про **осіб, які отримали відомості** про зареєстрований транспортний засіб, що йому належить». (Стосується офіційного каналу, не посередників — але задає етичну планку.)

### Воєнні обмеження (2022+)

- **Постанова КМУ № 1162 від 14.10.2022** «Деякі питання функціонування Єдиного державного реєстру транспортних засобів в умовах воєнного стану» (`https://zakon.rada.gov.ua/laws/show/1162-2022-п`) — доступ до ЄДРТЗ **зупиняється на визначених територіях**; при загрозі несанкціонованого доступу (окупація/оточення) адміністратор зобов'язаний **негайно зупинити й анулювати доступ користувачів протягом доби**.
- Ширший контекст: **Постанова КМУ № 209 від 06.03.2022** — реєстри Мін'юсту в умовах воєнного стану.
- 🔴 **`data.gov.ua` недоступний з цього середовища взагалі** (`http=000` через curl, помилка через WebFetch; `hsc.gov.ua` → 403). Схоже на геоблок для не-UA IP. **Тому статус набору «Відомості про транспортні засоби та їх власників» (`data.gov.ua/dataset/06779371-308f-42d7-895e-5a39833375f0`) я НЕ верифікував** — ні дату оновлення, ні наявність VIN/номерів, ні чи він живий. Хто перевірятиме — робити це з українського IP.

### Персональні дані

- Чинний закон — **№ 2297-VI «Про захист персональних даних» (2010)**. Законопроєкт **№ 8153** (адаптація до GDPR) зареєстровано 25.10.2022, прийнято за основу 20.12.2023, станом на **19.12.2025 — «Готується на друге читання»** (`https://itd.rada.gov.ua/billinfo/Bills/Card/40707`). **Законом не став.** Тобто GDPR-подібні штрафи/DPO поки не діють, але прийдуть — закладатись варто вже зараз.
- 🟢 **Дуже сприятливий факт:** ані Opendatabot `/v3/tech-passport`, ані baza-gai `/nomer/` **не повертають ПІБ/адресу власника**. Ми отримуємо **лише атрибути машини** (VIN, марка, рік, об'єм, пальне, колір). Це радикально знижує ризик: ми не обробляємо персональні дані власника — ми обробляємо характеристики авто.
  - Виняток, у який **не лізти**: Opendatabot `/v3/registered-cars/{code}` шукає авто **за ІПН людини**. Це вже справжні персональні дані. **Не використовувати ніколи.**

### Безпечна поза для hosted SaaS (рекомендована)

1. **Тільки своє авто.** Запит лише за номером, який користувач заявляє своїм. Явний чекбокс «Це моє авто / я маю право додати цей номер» **перед** запитом. Ніякої фічі «перевір будь-яке авто» — це вже інший продукт з іншим правовим профілем.
2. **Нічого не зберігати понад підтверджене.** Показати результат → користувач підтверджує/редагує → зберігаємо **тільки підтверджені поля**. Сира відповідь API не зберігається; кеш — короткий TTL і лише щоб не платити двічі за той самий номер.
3. **Анти-enumeration.** Rate limit на користувача (напр. 5 lookup/добу) + загальний ліміт. Без цього наш ключ = безкоштовний проксі до реєстру для сторонніх, і це і фінансовий, і правовий ризик.
4. **Ключ тільки на сервері.** Ендпоінт `POST /api/cars/lookup`, ключ в env. Ніколи не з клієнта — інакше ключ витягнуть з PWA-бандла за хвилину.
5. **Логи.** Не логувати номер разом із відповіддю; ротація.
6. **Прозорість.** Одна фраза в UI: «Дані з відкритого реєстру МВС через Opendatabot». Це і чесно, і знімає питання «звідки ви це взяли».

## 7. Оцінка вартості

**Модель використання:** «додати авто» — одноразова дія. Припущення: **1,2 авто/користувач**, **1,4 запити/авто** (промахи, друкарські помилки, повтори) → **~1,7 запити/користувач/рік**.

| | 100 користувачів | 1000 користувачів |
|---|---|---|
| Запитів/рік | ~170 | ~1 700 |
| Запитів/міс | ~14 | ~142 |
| **Opendatabot** (пакет 18 000/рік) | використано **~1%** квоти | використано **~9%** квоти |
| **Ціна Opendatabot** | **8 100 грн/рік = 675 грн/міс** | **8 100 грн/рік = 675 грн/міс** |
| **На користувача/міс** | **≈ 6,75 грн** | **≈ 0,68 грн** |
| **baza-gai** (free 1000/міс) | 0 грн — вкладаємось у 1,4% ліміту | 0 грн — вкладаємось у 14% ліміту |
| **Auto.dev** (free 1000/міс) | $0 | $0 |
| **vPIC / офлайн-декод року** | $0 | $0 |

**Головний висновок по грошах:** маржинальна вартість запиту (0,45 грн) **нерелевантна** — ми ніколи не наблизимось до квоти. Платимо **фіксовану підлогу пакета 8 100 грн/рік** незалежно від того, 100 у нас користувачів чи 1000. Тобто це рішення **«чи варта фіча 675 грн/міс»**, а не «скільки коштує масштабування». На 1000 користувачів це шум; на 100 — 6,75 грн/user/міс, що вже відчутно, якщо підписка коштує умовні 50 грн.

> 💡 Звідси й послідовність: **на малих числах безкоштовний ключ baza-gai покриває все з запасом**. Платити 8 100 грн/рік має сенс лише коли (а) фіча доведено потрібна, (б) потрібен `fuel` і договір, (в) є ФОП.

**Не верифіковано (не вгадую):** ціни baza-gai premium, ціни МТСБУ, платні тарифи Vincario. Курс грн→EUR не перераховував.

## 8. План B (якщо офіційних джерел нема / не купуємо ключ)

Він у нас майже готовий — і для ЄС-імпортів **працює краще за реєстр**.

1. **Ручний VIN + номер на `Car`.** Зараз у моделі (`backend/app/models.py:46-72`) нема **ні VIN, ні номера**. Це дешева Alembic-міграція. `docs/FEATURE-MAP.md` уже містить «VIN + номерний знак на Car (+ у PDF-звіті) — **S**».
2. **Офлайн-декод VIN → рік + марка.** Без API: WMI (перші 3 символи) + код модельного року (10-та позиція, ISO 3779). Емпірично підтверджено в §5, що саме ці поля виводяться зі структури. + валідація контрольної цифри (9-та позиція) — миттєво ловить друкарську помилку у 17 символах, що само по собі корисно.
3. **OCR техпаспорта** — 🟢 **найсильніша частина Плану B.** У нас **уже є**:
   - `backend/app/services/ocr.py` — tesseract, і `backend/app/services/ocr_llm.py` — **fallback на Gemini vision** (`gemini-flash-latest`);
   - `backend/app/routers/ocr.py` — `POST /api/ocr/scan`, JWT, ліміт 10 МБ;
   - фото-вкладення (`LogPhoto`, `backend/app/routers/photos.py`).

   **Чого нема:** OCR заточений **виключно під паливні чеки**, а фото чіпляються **тільки до `LogEntry`**, не до `Car`. Тобто робота реальна (новий промпт + car-level вкладення), але це **розширення робочої підсистеми, а не нова підсистема**.

   **Чому це стратегічно найкраще:**
   - 🟢 **Працює для щойно ввезених ЄС-авто, яких у реєстрі МВС ще/взагалі нема** — а це рівно наша цільова аудиторія (Golf 7 з Німеччини). Реєстр тут **безсилий**, техпаспорт — ні.
   - 🟢 **Нуль правових питань:** користувач фотографує **власний документ**. Ніякого третього-сторонного реєстру, ніяких персональних даних чужих людей.
   - 🟢 **Нуль вендор-локу і нуль фіксованої плати.** За оцінкою самого FEATURE-MAP — ~$0.01–0.03/скан, і платимо лише за факт.
   - 🟢 Техпаспорт містить **усе одразу**: VIN, рік, об'єм, пальне — тобто те, за чим ми йшли в реєстр.
4. **ОСЦПВ — вручну**, пресетом (P1/S, уже в беклозі, фронтенд уже його чекає). Це не компроміс — це **єдиний існуючий спосіб** (§1 ⚠️).

### 9.3. Що має бути в спеці (якщо робимо #3/#4)

**Схема / бекенд**
1. Alembic-міграція: `Car.vin` (`String(17)`, nullable, unique per user?), `Car.plate` (`String(10)`, nullable). Валідація: VIN — 17 символів, без `I/O/Q`, **контрольна цифра (9-та поз.)**; номер — regex українського формату + нормалізація (кирилиця↔латиниця гомогліфи! `ВС1234НХ` ≠ `BC1234HX` — обов'язково фолдити, у нас уже є `_LOOKALIKE_MAP` в `ocr.py` як приклад підходу).
2. `POST /api/cars/lookup` `{plate}` → `{vin, brand, model, year, capacity, fuel, kind, color}`. JWT. **Ключ тільки в env, ніколи на клієнті.**
3. Адаптер за інтерфейсом — `LookupProvider` з реалізаціями `BazaGaiProvider` / `OpendatabotProvider`, вибір через env. Без цього перехід #3→#4 = переписування.
4. **Rate limit: 5 lookup/користувач/добу** + глобальний денний ліміт + kill-switch через env.
5. Мапінг `fuel` → існуючий `Car.fuel_type` (`String(20)`): `БЕНЗИН`→petrol, `ДИЗЕЛЬ`→diesel, `ЕЛЕКТРО`→electric, **`ЕЛЕКТРО АБО БЕНЗИН`→hybrid** (реальне значення зі схеми Opendatabot — не забути, інакше впаде на гібридах). Невідоме значення → лишити поле порожнім, **не вгадувати**.
6. Мапінг `capacity` (см³, int) → `Car.engine` (free-text): напр. `1598` → `"1.6 (1598 см³)"`.
7. Обробка помилок: `404` (не знайдено) і `503` (сервіс недоступний) → **тиха деградація в ручну форму**. Онбординг **ніколи** не блокується падінням стороннього API.

**UI/UX**
8. Чекбокс «Це моє авто / я маю право додати цей номер» **до** запиту.
9. **Обов'язковий екран підтвердження**: жодне поле не пишеться в БД без явного підтвердження. Користувач може виправити будь-що (реєстр буває неправий — особливо `capacity` і `color`).
10. Підпис джерела: «Дані з відкритого реєстру МВС через {provider}».
11. Фолбек-копірайт для «VIN недоступний» (baza-gai, авто до 2021) — не показувати порожнечу як помилку.

**Приватність**

## 1. 🟢 data.gov.ua: датасет ЖИВИЙ і містить VIN — це найсильніший варіант

**«Відомості про транспортні засоби та їх власників»** (МВС), https://data.gov.ua/dataset/06779371-308f-42d7-895e-5a39833375f0 (перевірено 2026-07-15):

- `state=active`, **`metadata_modified=2026-07-02`**, оновлення **щомісяця**, 16 ресурсів 2013→2026
- Ліцензія **CC-BY — комерційне використання дозволене за умови атрибуції**
- **Поля перевірені за реальним файлом** (`reestrtz2026.zip` → `reestrtz02.07.2026.csv`, 292 МБ, UTF-8, роздільник `;`, 17 колонок):
  `PERSON; OPER_CODE; D_REG; DEP; BRAND; MODEL; VIN; MAKE_YEAR; COLOR; KIND; PURPOSE; BODY; FUEL; CAPACITY; POWER_KWT; OWN_WEIGHT; TOTAL_WEIGHT`
- **VIN є (повні 17 символів). Держномера немає. Особи власника немає** (`PERSON` = лише `P`/`J`)
- Тобто дає **саме те, що vPIC не дає**: MODEL, FUEL, CAPACITY (см³), POWER_KWT, BODY
- Датасет **номерних знаків мертвий**: 1 ресурс від 2022-01-17, оновлення припинені постановою КМУ №1151 від 10.09.2025. **Відкритого мапінгу номер→авто не існує** — це остаточно підтверджує зняття авто-ОСЦПВ і «додати за номером» безкоштовно.
- Застереження: origin віддає 522/таймаути з не-українських мереж (діставали через `r.jina.ai` + CKAN API); файли 36–113 МБ; схема дрейфує між роками (у файлі 2026 нема колонки регіону).

## 2. 🔴 vPIC на ЄС-VIN не просто слабкий — він **надлишковий**

Емпірично: `WVWZZZAUZGW123456` → заповнено **16 зі 154 полів**; `Model`, `FuelTypePrimary`, `DisplacementL`, `EngineCylinders`, `BodyClass` — **порожні**. Повертає `ErrorCode: "1,5,14,400"` (німецькі `Z`-філери вважає невалідними символами, контрольну цифру провалює).
Ключовий висновок: **все, що vPIC віддає на ЄС-VIN (Make, Manufacturer, PlantCountry, ModelYear), відтворюється ~50 рядками офлайн-таблиці WMI + позиція 10.** Мережевий виклик не потрібен взагалі. Таблицю модельного року перевірено емпірично: D→2013, E→2014, F→2015, G→2016, H→2017, J→2018, K→2019, L→2020.

## 3. ⚖️ Правова картина: воєнний стан ні до чого

- Автомобільний акт — **КМУ №1162 від 14.10.2022**: доступ до ЄДРТЗ призупинено **лише в межах окремих адміністративно-територіальних одиниць за переліком МВС**, ніколи не всією країною. «Дати відновлення» не існує — перемикача не було.
- Справжня стіна **довоєнна**: КМУ №260 від 25.03.2016 + зміна №1013 від 14.11.2018 **прибрала «власників» зі сфери** — дані власника закрили за 4 роки до вторгнення. Завершення воєнного стану цього не поверне.

## 4. 🟡 Дія.Шеринг — легальний consent-шлях (несподівано доступний)

КМУ **№423 від 13.04.2024**, п.3 прямо називає одержувачами «інших юридичних осіб та **фізичних осіб - підприємців**» — тобто **ФОП підходить**, не лише банки. Портал: https://integration.diia.gov.ua (публічних API-доків немає, вхід через `start@diia.gov.ua`).
